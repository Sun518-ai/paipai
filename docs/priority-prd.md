# PRD - 任务优先级功能

## 1. 概述与目标

为 paipai TodoMVC 添加任务优先级功能（P0/P1/P2/P3），让用户能够对任务进行紧急程度分类，并支持按优先级排序，紧急任务醒目显示。

## 2. 优先级定义

| 优先级 | 名称 | 颜色 | 图标 | 说明 |
|--------|------|------|------|------|
| P0 | 紧急 | 红色 | 🔴 | 最紧急，需要立刻处理 |
| P1 | 高 | 橙色 | 🟠 | 重要，近期需要完成 |
| P2 | 中 | 蓝色 | 🔵 | 普通优先级 |
| P3 | 低 | 灰色 | ⚪ | 可以稍后处理 |

- 默认优先级为 P3（低）
- 每个任务都可以单独设置优先级
- 支持随时修改优先级

## 3. 排序规则

1. **置顶（pinned）任务优先**
2. **同层级按优先级排序**：P0 > P1 > P2 > P3
3. **同优先级按创建时间排序**：新创建的在前

## 4. 优先级可视化

- 任务行的左侧，checkbox 旁边显示优先级图标
- P0 任务文字加粗
- P0 任务行左侧有一条红色边线（border-left accent）

## 5. 数据模型

```typescript
interface Todo {
  id: string;
  text: string;
  done: boolean;
  createdAt: number;
  pinned: boolean;
  priority: 'P0' | 'P1' | 'P2' | 'P3'; // 新增字段，默认 'P3'
}
```

## 6. 验收标准

- [ ] 每个任务可以设置 P0/P1/P2/P3 四个优先级
- [ ] P0 任务有红色醒目显示（图标 + 边线）
- [ ] 支持按优先级排序（P0 最前，P3 最后）
- [ ] 优先级修改自动持久化到 localStorage
- [ ] 新增任务默认优先级为 P3
- [ ] `npm run build` 通过
