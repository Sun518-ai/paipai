# 标签分类功能 PRD

## 1. 概述与目标

**功能名称：** 任务标签分类
**用户故事：** 作为用户，我希望给任务添加多个标签（如 #工作 #紧急 #学习），以便更好地组织和筛选任务。
**目标：** 在现有的 TodoMVC 页面上扩展标签功能，支持多标签管理、颜色自定义和按标签筛选。

## 2. 功能需求

### 2.1 核心功能

| 需求 | 描述 | 优先级 |
|------|------|--------|
| 多标签添加 | 每个任务可添加多个标签 | P0 |
| 标签颜色 | 标签支持自定义颜色 | P0 |
| 标签筛选 | 支持按标签筛选任务列表 | P0 |
| 标签管理 | 标签管理页面（增删改） | P1 |

### 2.2 用户交互流程

1. **添加任务时添加标签**：在任务输入框旁有标签选择器
2. **为已有任务添加标签**：点击任务上的「标签」按钮，打开标签选择浮层
3. **筛选任务**：点击标签筛选栏中的标签，只显示匹配的任务
4. **管理标签**：点击「标签管理」入口，进入标签管理页面

### 2.3 UI 组件

- **标签选择器**：显示所有标签，支持多选，选中态高亮
- **标签徽章**：任务右侧显示标签小徽章
- **标签管理弹窗/页面**：列表展示所有标签，支持增删改
- **筛选栏**：顶部显示所有标签按钮，点击筛选

### 2.4 标签数据结构

```typescript
interface Tag {
  id: string;
  name: string;       // 如 "工作"
  color: string;     // hex 颜色，如 "#3b82f6"
}
```

任务 Todo 接口扩展：
```typescript
interface Todo {
  // ... 现有字段
  tagIds: string[];   // 关联的标签 ID 数组
}
```

### 2.5 标签管理

- 默认标签：#工作（蓝）、#紧急（红）、#学习（绿）
- 支持新增标签（名称 + 颜色选择器）
- 支持编辑标签名称和颜色
- 支持删除标签（任务中关联自动清除）
- 标签持久化到 localStorage
