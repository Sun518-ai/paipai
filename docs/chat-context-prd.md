# TodoChat 多轮对话上下文 PRD

## 1. 背景

### 1.1 项目概述
「派派点子站」是一个为派派（孙识宇小朋友）打造的创意工具集合，其中的 TodoMVC 模块提供了待办事项管理功能。

### 1.2 当前问题
当前 TodoChat 仅支持单轮对话，用户每次刷新页面后对话上下文完全丢失，用户无法：
- 与 AI 继续之前的对话话题
- 在多轮交互中逐步完善待办事项
- 获得基于上下文的连贯建议

### 1.3 为什么需要多轮对话上下文
- **连贯性**：用户可以与 AI 进行多轮深入交流
- **上下文保持**：刷新页面后对话历史仍可追溯（可选）
- **更自然的交互**：类似真实对话，逐步明确需求

## 2. 需求

### 2.1 功能需求

#### 2.1.1 多轮对话支持
- 使用 Vercel AI SDK 的 `useChat` hook 管理消息状态
- 用户消息和 AI 回复均保存在 messages 数组中
- 对话历史在页面刷新前持续有效

#### 2.1.2 流式响应展示
- AI 响应通过 SSE 流式返回，逐字/逐句显示
- 打字机效果增强用户体验

#### 2.1.3 聊天 UI 交互
- 用户消息右对齐显示（气泡样式）
- AI 消息左对齐显示（气泡样式）
- 输入框支持 Enter 发送和按钮发送
- 加载状态显示动画

### 2.2 非功能需求

#### 2.2.1 性能
- 首次响应时间 < 500ms
- 流式输出延迟感知无明显卡顿

#### 2.2.2 兼容性
- 支持现代浏览器（Chrome、Firefox、Safari、Edge）
- 移动端适配（响应式布局）

## 3. 验收标准

### 3.1 功能验收

| 验收项 | 标准 | 测试方法 |
|--------|------|----------|
| 多轮对话 | 连续发送多条消息，AI 能理解上下文 | 发送多条相关消息，观察回复相关性 |
| 消息持久化（刷新前） | 刷新页面后对话历史清空（当前版本） | 刷新页面，观察消息是否清空 |
| 流式显示 | AI 响应以流式方式逐字显示 | 发送问题，观察是否为打字机效果 |
| 输入发送 | Enter 和按钮点击均可发送消息 | 实际测试两种发送方式 |
| 加载状态 | AI 响应时显示加载动画 | 观察 AI "typing..." 时的 UI |

### 3.2 UI 验收

| 验收项 | 标准 |
|--------|------|
| 用户消息 | 右对齐，气泡样式 |
| AI 消息 | 左对齐，气泡样式 |
| 输入框 | 底部固定，聚焦时有视觉反馈 |
| 加载动画 | 流式输出时有"正在输入..."指示 |
