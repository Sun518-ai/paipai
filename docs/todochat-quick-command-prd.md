# TodoChat 快速执行通路 PRD

## 1. 背景与目标

### 背景
- TodoMVC 目前支持 AI 助手（TodoChat），用户需要手动点击展开/收起
- 对于偶尔需要 AI 辅助的场景，用户希望有更快的"直达通道"
- 降低使用门槛，让 AI 触手可及

### 目标
- 在 Todo 输入框中，输入 `+` 开头的文字直接触发 AI 助手对话
- 无需额外操作，实现"一键唤起"体验
- 极简快速通道，降低认知负担

---

## 2. 功能详细描述

### 核心行为
- **触发条件**：用户在 Todo 输入框中输入文字，第一个字符为 `+`
- **执行动作**：
  1. 不创建 Todo 项
  2. 自动展开/打开 TodoChat
  3. 将 `+` 后面的内容作为用户第一条消息传入 AI 助手
- **边界处理**：
  - `+` 后面没有内容（如只输入 `+`）：不触发，直接清空输入框
  - 普通文字（不以 `+` 开头）：正常创建 Todo

### 用户交互流程
```
用户输入 "+ 今天天气怎么样？"
    ↓
检测到 + 开头
    ↓
清空输入框
    ↓
展开 TodoChat
    ↓
自动发送 "今天天气怎么样？" 作为第一条消息
    ↓
AI 助手响应
```

---

## 3. 用户交互流程（详细）

### 场景一：正常创建 Todo
1. 用户在输入框输入：`买牛奶`
2. 按回车或点击添加
3. Todo 列表新增 "买牛奶"

### 场景二：快速唤起 AI
1. 用户在输入框输入：`+ 今天天气怎么样？`
2. 按回车或点击添加
3. 输入框清空
4. TodoChat 自动展开（如未展开）
5. 用户消息 "今天天气怎么样？" 自动出现在对话中
6. AI 开始流式响应

### 场景三：只输入 +
1. 用户在输入框输入：`+`
2. 按回车
3. 输入框清空（不触发任何操作）

## 4. 验收标准

### 功能验收
- [ ] 输入 `+` 开头的文字后，按回车不创建 Todo
- [ ] 输入 `+` 开头的文字后，自动展开 TodoChat（如已折叠）
- [ ] `+` 后面的内容作为第一条用户消息出现在 AI 对话中
- [ ] AI 正常响应流式输出
- [ ] 只输入 `+` 时，不触发任何操作，输入框清空
- [ ] 普通文字输入（不以 `+` 开头）仍然正常创建 Todo

### UI/UX 验收
- [ ] TodoChat 展开时，页面有正确的视觉反馈
- [ ] AI 消息正确显示在对话区域
- [ ] 输入框在触发后正确清空

### 技术验收
- [ ] `npm run build` 通过
- [ ] 无 TypeScript 错误
- [ ] 复用现有代码模式，无重复代码

## 5. 非功能要求

### 兼容性
- 支持现有 TodoMVC 所有功能（优先级、置顶、筛选等）
- 与深色模式兼容

### 性能
- 检测逻辑在输入事件中完成，无额外延迟
- TodoChat 组件按需加载（已有 dynamic import）
