# PRD: TodoChat 删除确认对话框

## 1. 背景

当前 TodoMVC 页面中的删除操作（🗑️ 按钮）直接删除任务，无任何确认步骤。用户可能因误触导致数据丢失，用户体验不佳，存在误删风险。

## 2. 需求描述

在删除任务前，弹出确认对话框，让用户确认是否真的要删除该任务，防止误删。

## 3. 用户场景

- 用户点击任务的 🗑️ 删除按钮
- 系统弹出确认对话框，显示任务内容摘要
- 用户选择「确认」则删除任务，选择「取消」则保持不变

## 4. 验收标准

1. **AC1**: 点击删除按钮弹出确认对话框
2. **AC2**: 对话框显示任务文本摘要（最多 30 字符）
3. **AC3**: 点击「确认」后执行删除，任务从列表消失
4. **AC4**: 点击「取消」或对话框背景，不删除任务，对话框关闭
5. **AC5**: 支持键盘操作（Enter 确认，Escape 取消）
6. **AC6**: 对话框样式与项目 dark mode 兼容
7. **AC7**: 确认按钮样式醒目（红色），取消按钮为灰色

## 5. 非功能要求

- 交互延迟 < 100ms
- 对话框关闭时无残留动画
- 移动端可用（点击区域 >= 44px）
